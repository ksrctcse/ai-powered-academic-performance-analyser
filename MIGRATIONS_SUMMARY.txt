## Migration System Features

### Command Overview
```bash
python run_migrations.py status              # View status
python run_migrations.py                     # Run all pending
python run_migrations.py up MIGRATION_NAME   # Run specific
python run_migrations.py rollback            # Undo last
python run_migrations.py --verbose           # Detailed output
```

### Automatic Tracking
```json
# migration_history.json (created automatically)
[
  {
    "name": "004_initial_schema_complete",
    "executed_at": "2026-03-24T12:30:00",
    "direction": "up"
  }
]
```

### Full Reversibility
```bash
# Run forward
python run_migrations.py up 004_initial_schema_complete

# Roll back  
python run_migrations.py down 004_initial_schema_complete
```

## Development Workflow

### Adding New Tables
1. Create migration file: `005_my_feature.py`
2. Implement upgrade() with CREATE TABLE
3. Implement downgrade() with DROP TABLE
4. Run: `python run_migrations.py up 005_my_feature`
5. Test the change
6. Commit both .py file and updated migration_history.json

### Example New Migration
```bash
cat > backend/migrations/005_my_table.py << 'EOF'
from sqlalchemy import text
from app.database.session import engine

def upgrade():
    with engine.connect() as conn:
        conn.execute(text("CREATE TABLE my_table (id SERIAL PRIMARY KEY)"))
        conn.commit()

def downgrade():
    with engine.connect() as conn:
        conn.execute(text("DROP TABLE my_table"))
        conn.commit()
EOF

python run_migrations.py up 005_my_table
```
